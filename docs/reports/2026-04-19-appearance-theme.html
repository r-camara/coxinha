PR overview

Adds a Settings > Appearance section with an Auto / Light / Dark theme radio. `auto` keeps the existing prefers-color-scheme follow with a live OS listener; explicit light / dark ignore the OS and stay sticky. Preference persists in localStorage (coxinha.themePref) and broadcasts via CustomEvent('coxinha:theme-pref-changed') so App.tsx re-applies without a reload. config.toml is intentionally untouched — theme is a per-install UI affordance, not a vault setting (and it avoids a round-trip to disk on every toggle).
